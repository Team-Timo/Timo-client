# timo-orchestrate

## 트리거

- 하나의 요청이 여러 스킬을 순서대로 필요로 하는 경우
- "이 기능 처음부터 끝까지 만들어줘"

## 참조

- `AGENTS.md` → 스킬 라우팅 테이블
- `docs/agent/skill-index.md` → 전체 스킬 목록

## 워크플로우

### Phase 1 — 요청 분해
사용자 요청을 읽고 필요한 스킬 목록과 실행 순서를 도출한다.

예시: "소셜 로그인 페이지 만들고 커밋까지"
```
1. timo-issue   → 이슈 생성 및 브랜치명 제안
2. timo-page    → 페이지 구현
3. timo-review  → 코드 리뷰
4. timo-verify  → PR 전 검증
5. timo-commit  → 커밋
6. timo-pr      → PR 작성
```

### Phase 2 — 계획 확인
실행 순서와 각 스킬 역할을 사용자에게 보여주고 승인받는다.

### Phase 3 — 순차 실행
각 스킬을 순서대로 실행한다.  
블로커 발생 시 중단하고 사용자 확인을 요청한다.

### Phase 4 — 완료 보고
전체 실행 결과를 요약한다.
